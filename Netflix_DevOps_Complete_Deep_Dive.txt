📦 WHAT IS A DOCKER IMAGE? (Step-by-Step Explanation)

STEP 1: THE BASIC CONCEPT
```
Docker Image = Complete Package containing:
- Operating System (Linux)
- Application Code (Netflix video player)
- Dependencies (libraries, packages)
- Configuration (settings, environment)
- Runtime (Node.js, Python, Java)

Think of it as a ZIP file containing everything needed to run Netflix!
```

STEP 3: HOW NETFLIX CREATES DOCKER IMAGES (Complete Process)
```
Netflix Developer writes Dockerfile:

# Start with Node.js base image
FROM node:16-alpine
# Downloads Alpine Linux (5MB) + Node.js (50MB)
# Alpine is chosen because it's lightweight and secure

# Set working directory inside container
WORKDIR /app
# Creates /app directory and sets it as current directory
# All subsequent commands run from /app

# Copy package files first (for better caching)
COPY package.json package-lock.json ./
# Copies only dependency files initially
# This enables Docker layer caching for faster builds

# Install dependencies
RUN npm ci --only=production
# Downloads and installs all packages listed in package.json
# --only=production skips development dependencies
# Creates node_modules/ directory with 200+ packages

# Copy application source code
COPY src/ ./src/
COPY public/ ./public/
COPY config/ ./config/
# Copies all application files to container
# Done after npm install for better caching

# Create non-root user for security
RUN addgroup -g 1001 -S nodejs
RUN adduser -S nextjs -u 1001
USER nextjs
# Security best practice - never run as root user
# Creates dedicated user with limited permissions

# Expose port 3000
EXPOSE 3000
# Documents that this app listens on port 3000
# Doesn't actually publish the port (done at runtime)

# Set environment variables
ENV NODE_ENV=production
ENV PORT=3000
ENV LOG_LEVEL=info
# Configures runtime environment
# These can be overridden when running container

# Health check command
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD curl -f http://localhost:3000/health || exit 1
# Kubernetes uses this to check if container is healthy
# Runs every 30 seconds, fails after 3 consecutive failures

# Start command
CMD ["npm", "start"]
# Default command when container starts
# Runs "npm start" which executes "node video-player.js"
```

STEP 5: IMAGE LAYERS EXPLAINED (Very Important Concept!)
```
Why Docker uses layers (like onion layers):

BENEFIT 1: Caching for Faster Builds
Scenario: Developer changes one line in video-player.js

Build 1 (First time):
├── Layer 1: Alpine Linux (5 MB) - Downloaded
├── Layer 2: Node.js (50 MB) - Downloaded  
├── Layer 3: Dependencies (100 MB) - Downloaded
├── Layer 4: App code (20 MB) - Downloaded
└── Total time: 8 minutes

Build 2 (After code change):
├── Layer 1: Alpine Linux - CACHED (0 seconds)
├── Layer 2: Node.js - CACHED (0 seconds)
├── Layer 3: Dependencies - CACHED (0 seconds)
├── Layer 4: App code - REBUILT (30 seconds)
└── Total time: 30 seconds (16x faster!)

BENEFIT 2: Storage Efficiency
Netflix has 1000 different services, all using node:16-alpine:
├── Without layers: 1000 × 55 MB = 55 GB storage
├── With layers: 55 MB × 1 = 55 MB storage
└── Space saved: 54.945 GB (99.9% savings!)

BENEFIT 3: Network Efficiency
Pushing updated image to registry:
├── Layer 1-3: Already exist in registry (skipped)
├── Layer 4: Only changed layer uploaded (20 MB)
└── Upload time: 2 minutes instead of 15 minutes
```

🐳 WHAT IS A DOCKER CONTAINER? (Step-by-Step Explanation)

STEP 1: THE BASIC CONCEPT
```
Docker Container = Running instance of an image
- Image is like a program file (.exe on Windows)
- Container is like running that program in memory
- One image can create thousands of containers
- Each container is completely isolated from others
- Containers share the host OS kernel (lightweight)
```

STEP 2: CREATING CONTAINER FROM IMAGE (Complete Process)
```
Netflix runs:

   ├── Docker checks if netflix-video-player:v3.0.1 exists locally

2. CONTAINER CREATION
   ├── Generates unique container ID: abc123def456789
```

```
Kubernetes Cluster = Group of computers working together

- Factory Manager (Master Node)

- Work Teams (Pods)
- Communication System (Services)
```

```
   ├── kubectl creates Service object in etcd
```

```
WHAT IS INGRESS?
Ingress = HTTP/HTTPS entry point into cluster
- Manages external access to services
- Provides SSL termination
- Enables path-based and host-based routing
- Single entry point for multiple services

│   ├── netflix.com/api/recommendations → recommendation-service:80

TRAFFIC FLOW THROUGH INGRESS:

9. User sees Netflix homepage
```
