# Advanced Shell Scripting – Node Health Monitoring

## 1. Goal of This Lesson

The instructor is teaching how to create a **shell script that checks the health of a server (node)**.

In real DevOps work, engineers constantly check:

* Disk usage
* Memory usage
* CPU usage
* Running processes

Instead of checking manually every time, we write a **script that prints everything automatically**.

> "Check the health of the server."

# 3. Commands Used to Check Node Health

The instructor reviewed commands from the previous lesson.

These are **very important Linux commands**.

---

# 4. Disk Usage Command

### Command

```bash
df -h
```

### Meaning

**df = disk filesystem**

Shows how much disk storage is used and available.

### Example Output

```
Filesystem      Size  Used Avail Use%
/dev/xvda1       20G   10G   10G  50%
```

Meaning

| Column | Meaning         |
| ------ | --------------- |
| Size   | Total disk      |
| Used   | Used space      |
| Avail  | Free space      |
| Use%   | Percentage used |

DevOps engineers use this to check if disk is full.

---

# 5. Memory Command

### Command

```
free -h
```

### Meaning

Shows RAM usage.

Example output:

```
total   used   free
2GB     500MB  1.5GB
```

This helps detect:

* memory leaks
* high RAM usage
* server crashes

---

# 6. CPU Command

### Command

```
nproc
```

### Meaning

Shows **number of CPU cores**.

Example:

```
2
```

Meaning the machine has **2 CPU cores**.

---

# 7. Process Monitoring Command

### Command

```
top
```

### Meaning

Shows **live processes running in the system**.

Information shown:

* CPU usage
* RAM usage
* running processes
* sleeping processes
* stopped processes

Example info:

```
PID USER CPU MEM COMMAND
```

This command is used when a server becomes slow.

---

# 8. Why DevOps Engineers Check These

Before debugging a server issue, engineers check:

1️⃣ Disk space
2️⃣ Memory usage
3️⃣ CPU usage
4️⃣ Running processes

Example problem:

Server slow.

Possible reasons:

* Disk full
* RAM full
* CPU overloaded
* Too many processes

Instead of running commands manually:

# 12. Important Term: Shell Script

### Definition

A **shell script** is a file that contains Linux commands executed automatically.

Example:

```
script.sh
```

Extension:

```
.sh
```

---

# 13. First Line of Every Shell Script

### Shebang

First line should always be:

```
#!/bin/bash
```

### Meaning

This tells Linux:

> Run this script using the **Bash shell**.

---

# 14. What is Bash?

### Definition

**Bash = Bourne Again Shell**

It is a **command interpreter used in Linux**.

It reads commands and executes them.

Example:

When you type:

```
ls
```

Bash executes the command.

---

# 15. Example Script Structure

Create script.

```
nano node_health.sh
```

Add this:

```bash
#!/bin/bash

echo "Disk Usage"
df -h

echo "Memory Usage"
free -h

echo "CPU Cores"
nproc

echo "Running Processes"
top
```
