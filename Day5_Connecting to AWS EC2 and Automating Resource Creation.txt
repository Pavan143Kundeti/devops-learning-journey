3. Select the instance
4. Click **Connect**
5. AWS opens a browser terminal

Now you are inside the machine.

Example command:

```

```

**touch command**

Creates an empty file.

Example:

```
touch file1
```

Then check using:

```
ls
```

Definition:

**ls command**

Lists all files in the folder.

# 7. Important Term: SSH

Definition:

**SSH (Secure Shell)**

A protocol that allows you to securely connect to another computer.

Simple meaning:

SSH lets your laptop **control a remote server securely**.

---

Example command:

```
ssh ubuntu@3.89.45.22
```

Meaning:

```
ssh     → connection protocol
ubuntu  → username
3.89.45.22 → server IP
```

But this command will fail.

Why?

Because AWS requires **key authentication**.

---

# 8. What Is a Key Pair

Definition:

**Key Pair**

A security system used by AWS to allow login.

Two parts exist:

1. **Public Key** → stored in the server
2. **Private Key (.pem file)** → stored on your laptop

Only when both match, login works.

---

Real world example:

Think of it like a **house lock**.

Lock on the door = Public Key
Key in your pocket = Private Key

Without the key, the door will not open.

---

# 9. Correct SSH Command

You must provide the key file.

Command:

```
ssh -i test11.pem ubuntu@3.89.45.22
```

Meaning:

```
-i → identity file
test11.pem → private key
ubuntu → username
3.89.45.22 → server IP
```

---

# 10. Permission Error Problem

When you try this, you may see error:

```
Permissions are too open
```

Why?

Because the **pem file is sensitive**.

If everyone can access it, anyone could login to your server.

---

# Fix the Permission

Use command:

```
chmod 600 test11.pem
```

Definition:

**chmod**

Changes file permissions.

Meaning of 600:

```
Owner → read + write
Others → no access
```

After that run again:

```
ssh -i test11.pem ubuntu@IP
```

Now login succeeds.

# 12. Instance Billing Concept

AWS charges money while the instance is **running**.

States of EC2:

| State      | Meaning                        |
| ---------- | ------------------------------ |
| Running    | Server active, billing running |
| Stopped    | Server paused                  |
| Terminated | Server deleted                 |

Best practice:

Stop server when not using.

Example:

Testing server used only 2 hours.

Stop it to save cost.

# 17. Authentication Using Access Keys

Go to:

AWS Console → Profile → Security Credentials

Create:

**Access Key**

You will get two values:

```
Access Key ID
Secret Access Key
```

---

Definition:

**Access Key**

Credential used by programs to access AWS.

Think of it like:

```
username + password
for software access
```

---

# 18. Configure AWS CLI

Run:

```
aws configure
```

You will enter:

```
Access Key ID
Secret Access Key
Region
Output format
```

Example:

```
Access Key: XXXXX
Secret Key: XXXXX
Region: us-east-1
Output: json
```

Now CLI is connected to your AWS account.

# 23. Another Automation Method — Python

AWS provides Python library:

**boto3**

Definition:

**boto3**

Python SDK that allows programs to control AWS services.

Example Python script:

```
import boto3

ec2 = boto3.client('ec2')
response = ec2.describe_instances()

print(response)
```

This script lists EC2 instances.

So developers can build automation scripts.

# 1. First Concept: SSH (The Actual Technology)

**SSH = Secure Shell**

Definition
A network protocol used to **securely connect to another computer over the internet**.

Simple meaning

Your laptop → connects → remote server.

Example:

```
ssh ubuntu@3.89.45.22
```

You are telling your laptop:

"Connect to the server at this IP and login as user ubuntu."

Important point:

SSH is **not a software**.

It is a **communication method**.

Like:

* HTTP for websites
* FTP for file transfer
* SSH for remote login

---

A terminal is simply a **command window**.

### Windows terminals

* Git Bash

# 3. Then Why Does MobaXterm Exist?

Because older Windows systems **did not have good SSH tools**.

So tools were created.

One popular tool is:

**MobaXterm**

Definition

MobaXterm is a **terminal software with built-in SSH and file transfer tools**.

Think of it like:

A **remote server control panel**.

It gives:

* SSH connection
* terminal
* file browser
* SFTP transfer
* session saving

# 4. When to Use Each Option

Now the important clarity.

### Option 1 — Normal Terminal (PowerShell / CMD)

Used when you want **direct command control**.

# 5. Where Does AWS CLI Fit?

This is **completely different**.

AWS CLI is **not for connecting to servers**.

AWS CLI is used to **control AWS services**.

Example tasks:

Create EC2

```
aws ec2 run-instances
```

Create S3 bucket

```
aws s3 mb s3://mybucket
```

List servers

```
aws ec2 describe-instances
```

Delete resources

```
aws ec2 terminate-instances
```

So AWS CLI is used to **manage cloud infrastructure**.
