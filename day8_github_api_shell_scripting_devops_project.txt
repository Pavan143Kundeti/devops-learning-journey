# 2. How DevOps Talks to Applications

There are **2 ways** to communicate with software.

### Method 1

**User Interface (UI)**

Example:

```
Browser
→ github.com
→ click buttons
```

Example:

```
GitHub → open repo → settings → collaborators
```

---

### Method 2

**API**

Instead of clicking UI, you send a request.

Example:

```
script → API → GitHub → response
```

---

# Definition

### API

API = Application Programming Interface

Meaning:

```
A way for software to talk to another software.
```

Example:

```
Your script → GitHub API
```

# 5. What is `curl`

### Definition

`curl`

A command-line tool used to send HTTP requests to APIs.

Example:

```
curl https://api.github.com
```

It fetches data.

---

# 6. What is `jq`

### Definition

`jq`

A tool used to **extract fields from JSON data**.

Example JSON:

```
{
"name": "pavan",
"role": "developer"
}
```

Extract name:

```
jq '.name'
```

Output:

```
pavan
```

---

# 7. Why GitHub Token is Needed

Normally you login using:

```
username
password
```

But APIs use:

```
token
```

---

# Definition

### GitHub Personal Access Token

A secret key used to authenticate API requests.

Example:

```
ghp_xxxxxxx
```

Think of it like:

```
API password
```

Never share it.

# 12. Inside the Script (Simple Logic)

Step 1

Get API URL

```
https://api.github.com/repos/<owner>/<repo>/collaborators
```

---

Step 2

Call API using curl

```
curl -u username:token URL
```

---

Step 3

GitHub returns JSON

Example:

```
[
 {
  "login": "moit",
  "permissions": {
      "admin": false
  }
 }
]
```

---

Step 4

Use jq to filter

Example:

```
jq '.[] | select(.permissions.pull==true) | .login'
```

Meaning:

```
show users with read access
```

---

# 13. Why Only "moit" Appeared

Because:

```
admin users ignored
```

Script logic:

```
only show non-admin users
```

Admin users:

```
devops team
```

Collaborators:

```
developers
```

# 14. Why Kubernetes Repo Failed

You tried:

```
./list-users.sh kubernetes kubernetes
```

Error happened.

Reason:

```
You don't have permission
```

GitHub API respects permissions.

If you cannot see settings in UI → API also blocks.
