# 8. How Automation Works

Cloud providers give something called:

**API**

---

# 9. What is an API (Super Simple Explanation)

API = A way for programs to talk to a service.

Example real life:

You go to restaurant.

You don’t go into the kitchen.

You talk to **waiter**.

Waiter talks to kitchen.

Kitchen prepares food.

Food comes back to you.

---

### Mapping

```
You → Script
Waiter → API
Kitchen → Cloud server
Food → Virtual Machine
```

---

# 10. AWS API Example

AWS provides:

```
EC2 API
```

Your script sends request:

```
Create VM
OS: Ubuntu
RAM: 4GB
CPU: 2
```

AWS API checks:

3 things.

---

### 1️⃣ Request must be valid

Example correct format.

```
OS=Ubuntu
CPU=2
RAM=4GB
```

---

### 2️⃣ User must be authenticated

Means:

You must prove **who you are**.

Example:

```
AWS Access Key
AWS Secret Key
```

---

### 3️⃣ User must be authorized

Example:

Maybe you have AWS account.

But you may **not have permission to create servers**.

Then AWS will reject.

# 12. Tools Used for Automation

DevOps engineers use multiple tools.

---

# Method 1 — AWS CLI

CLI means:

**Command Line Interface**

Instead of clicking UI you type commands.

Example:

```
aws ec2 run-instances
```

This command creates VM.

---

### Real Example

```
aws ec2 run-instances \
--image-id ami-12345 \
--count 10 \
--instance-type t2.micro
```

Result:

```
10 virtual machines created
```

---

# Method 2 — Direct API

If you are a programmer.

---

# 2. Meaning of This SSH Command

```
ssh -i my-key.pem ubuntu@3.89.45.22
```

Break it into parts.

### ssh

**Definition**

SSH = Secure Shell

**Meaning**

A secure way to **connect to another computer over the internet**.

Example:

You sit in India and connect to a server in **AWS US region**.

You can control that server from your terminal.

Just like sitting in front of it.

---

### -i

`-i` means **identity file**

It tells SSH:

> use this private key to login

---

### my-key.pem

This is a **private key file**.

AWS gives this file when you create a server.

Purpose:

Authentication.

Instead of password, AWS uses **key-based login**.

Example file:

```
my-key.pem
```

You download it once.

If you lose it → you cannot login.

---

### ubuntu

This is the **username of the server OS**.

Different OS have different usernames.

Examples:

Ubuntu server

```
ubuntu
```

Amazon Linux

```
ec2-user
```

CentOS

```
centos
```

---

### 3.89.45.22

This is the **Public IP Address** of the server.

Definition:

Public IP = internet address of the server.

Example:

```
3.89.45.22
```

Your laptop connects to this IP.

---

# Real Life Example

Think like this.

Your laptop = your house

AWS server = office computer

SSH = remote control access

The command means:

> Use my key to securely login to that remote computer.

---

# What Happens After Running the Command

When you run:

```
ssh -i my-key.pem ubuntu@3.89.45.22
```

The system will:

1. Connect to the server IP
2. Authenticate using the key
3. Open remote terminal

Now your terminal becomes **server terminal**.

Example prompt:

```
ubuntu@ip-172-31-24-15:~$
```

Now you can run commands like:

```
ls
mkdir
docker
git
npm
python
```

You are controlling the **cloud server**.

# 3. Infrastructure as Code (IaC)

You asked this also.

This is a **very important DevOps concept**.

---

## Definition

Infrastructure as Code means:

> Creating and managing servers, networks, and cloud resources using **code instead of manual setup**.

---

## Old Way (Manual)

Engineer logs into AWS console.

Then manually:

1. Create server
2. Configure network
3. Install software
4. Setup storage

Every time repeating.

Problems:

* Slow
* Human mistakes
* Hard to reproduce

---

## Modern Way (Infrastructure as Code)

Instead of manual steps, we write **configuration code**.

Example:

```
create server
RAM = 8GB
CPU = 4
OS = Ubuntu
region = Mumbai
```

Run the code.

AWS automatically creates everything.

---

## Popular IaC Tools

DevOps engineers use:

* Terraform
* AWS CloudFormation
* Ansible
* Pulumi

---

## Example (Terraform)

Example code:

```
resource "aws_instance" "web_server" {
  ami           = "ami-123456"
  instance_type = "t2.micro"
}
```

Run:

```
terraform apply
```

Boom.

Server is created automatically.

---

## Real Life Example

Manual way:

Cooking food by hand every day.

Infrastructure as Code:

Saving **recipe**.

Whenever needed → run recipe → food ready.
