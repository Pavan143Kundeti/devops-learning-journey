# 2. Important Term: Configuration

**Configuration means the setup of a server.**

It includes things like:

```
software installed
operating system
network settings
users
security rules
services running
```

Example configuration of a server:

```
OS: Ubuntu 22
Software: nginx
Database: mysql
User: devops
Port: 80 open
Firewall enabled
```

This entire setup is called **server configuration**.

# 3. Important Term: Configuration Management

Configuration Management means:

**controlling and maintaining server configurations automatically.**

Tasks include:

```
install software
update software
remove software
apply security patches
configure services
create users
change system settings
```

Instead of doing this manually on each server, tools do it **automatically**.

# 4. Important Term: Software Patch

A **patch** is a small update released by software developers.

Purpose of patches:

```

fix security vulnerabilities
improve performance
add small features
```

Example:

```

Apache releases patch
Linux kernel releases patch
```

When a patch comes, servers must be updated.

# 11. Problem with Scripts

```

```

```

```

Scripts must handle many conditions.

---

### Problem 2: Script Complexity

Script becomes large.

Example logic:

```

 run apt

if centos
 run yum

 run apk
```

Hard to maintain.

---

### Problem 3: Scaling

Script must run on thousands of servers.

Example:

```

ssh server
run command
```

Not reliable.

# 19. Push Model vs Pull Model

Important interview concept.

---

## Puppet → Pull Model

Servers ask master for updates.

Flow:

```
server → puppet master → get configuration
```

Servers periodically check for updates.

---

## Ansible → Push Model

Engineer pushes configuration.

Flow:

```
laptop → servers
```

Example command:

```
ansible-playbook install_nginx.yml
```

All servers update instantly.

---

# 20. Architecture Difference

### Puppet

Uses **Master–Slave architecture**.

Structure:

```
Puppet Master
   |
Agents installed on servers
```

Each server runs puppet agent.

Extra configuration needed.

---

### Ansible

Uses **Agentless architecture**.

No software required on servers.

Requirement:

```
SSH access
```

That’s all.

---

# 21. Inventory File

Ansible needs a file listing servers.

Example inventory:

```
[servers]
54.11.21.10
3.91.44.22
18.220.12.55
```

These are server IP addresses.

Ansible reads this file to know which servers to manage.

---

# 22. Passwordless Authentication

Ansible must connect to servers automatically.

Solution:

SSH keys.

Commands:

Generate key:

```
ssh-keygen
```

Copy key to server:

```
ssh-copy-id ubuntu@server-ip
```

Now login without password.
