# Concepts Covered

## 1. EC2 (Elastic Compute Cloud)

Definition:

EC2 is a cloud service that provides **virtual servers** on AWS.

You can install software, run applications, and host websites.

Example:

Instead of running a website on your laptop, you run it on an **EC2 server**.

Benefits:

* no hardware maintenance
* scalable
* accessible from internet
* high performance servers

---

# 2. IAM (Identity and Access Management)

Definition:

IAM is used to **control who can access AWS resources**.

Example in companies:

Different team members get different permissions.

Example roles:

Developer → EC2 access only
DevOps → EC2 + S3 + IAM
Security team → monitoring access

Best practice:

Never use **Root account** for daily work.

Always create **IAM users**.

---

# 3. SSH (Secure Shell)

Definition:

SSH is a protocol used to **securely connect to a remote server**.

Example:

You connect your laptop terminal to an EC2 server.

Command format:

```
ssh -i key.pem username@server-ip
```

Example:

```
ssh -i demo.pem ubuntu@54.252.219.95
```

---

# 4. Environment Variables (.env file)

Definition:

A file used to store **sensitive configuration values**.

Example:

```
API_KEYS
DATABASE_PASSWORD
PAYMENT_SECRET
PORT
DOMAIN
```

Why use it?

Security.

We do not store secrets directly inside source code.

---

# 5. NodeJS

Definition:

NodeJS allows running **JavaScript outside the browser**.

Used for:

* backend APIs
* servers
* microservices

---

# 6. NPM (Node Package Manager)

Definition:

NPM installs libraries required for a NodeJS application.

Example libraries:

* express
* stripe
* dotenv

Command:

```
npm install
```

# Step 1 — Clone GitHub Repository

Command:

```
git clone <repository_url>
```

Example:

```
git clone https://github.com/project/aws-session.git
```

Purpose:

Download project to your system.

# Step 2 — Enter Project Folder

Command:

```
cd aws-session
```

Meaning:

`cd` = change directory.

---

# Step 3 — Create Environment File

Command:

```
touch .env
```

Meaning:

Creates a hidden file.

---

# Step 4 — Install Dependencies

Command:

```
npm install
```

What it does:

Installs all required packages.

These packages go into:

```
node_modules/
```

---

# Step 5 — Run Application

Command:

```
npm run start
```

Server starts.

Example output:

```
Server running on port 3000
```

Test locally:

```
http://localhost:3000
```

---

# Step 6 — Create IAM User

Go to:

AWS Console → IAM → Users → Add User

Steps:

1. give username
2. enable console access
3. set password
4. assign permissions

For demo:

```
AdministratorAccess
```

In companies:

Use **least privilege principle**.

---

# Step 7 — Launch EC2 Instance

Go to:

```
AWS → EC2 → Launch Instance
```

Configuration used in demo:

Name:

```
demo-nodejs
```

AMI:

```
Ubuntu
```

Instance Type:

```
t2.micro
```

Reason:

Free tier.

---

# Step 8 — Create Key Pair

Used for SSH authentication.

File downloaded:

```
demo.pem
```

Important rule:

Never share this key.

---

# Step 9 — Connect to EC2 using SSH

Move to Downloads folder:

```
cd Downloads
```

Change key permissions:

```
chmod 400 demo.pem
```

Explanation:

Only owner can read the key.

Then connect:

```
ssh -i demo.pem ubuntu@<public-ip>
```

Example:

```
ssh -i demo.pem ubuntu@54.252.219.95
```

Now you are inside EC2 server.

Prompt looks like:

```
ubuntu@ip-172-31-23-36
```

---

# Step 16 — Run Application

---

# Step 17 — Application Not Accessible

Trying:

```
http://<public-ip>:3000
```

Result:

```
Site can't be reached
```

Reason:

Port not exposed.

---

# Step 18 — Open Security Group

Go to:

```
EC2
→ Instance
→ Security
→ Security Groups
```

Edit inbound rules.

Add rule:

Type:

```
Custom TCP
```

Port:

```
3000
```

Source:

```
0.0.0.0/0
```

Meaning:

Allow access from anywhere.

---

# Step 19 — Access Application

Now open:

```
http://<public-ip>:3000
```

Application loads successfully.

# Important DevOps Lessons from This Project

1. Always test application locally first.

2. Never deploy directly without testing.

3. Use IAM users instead of root.

4. Secure servers using SSH keys.

5. Expose only necessary ports.

6. Use environment variables for secrets.
